## Fixes appliqués

### 1. IDLE 1 (0xF4E1) catch-all NOP bug — `calypso_c54x.c:4007`

**Symptôme** : DSP traverse le boot stub 0x0000-0x007F (filled avec IDLE 1)
sans jamais s'arrêter, `s->idle` reste `false`, IRQ handler ne prend pas
le branch IDLE → INT3 jamais dispatché → INTM stuck.

**Root cause** : un catch-all `op >= 0xF4E0 && op <= 0xF4FF` traitait
**toutes** les opcodes du range comme NOP (sauf FRET 0xF4E4 et RETE 0xF4EB).
0xF4E1 (IDLE 1) tombait dedans et était silencieusement avalé. L'IDLE
handler dédié (ligne ~4058) n'était jamais atteint.

Le commentaire historique disait "RSBX/SSBX status bits" — c'est faux,
RSBX/SSBX sont à 0xF4B0/0xF5B0 (mask 0xFDF0), hors du range.

**Fix** : ajouter `&& op != 0xF4E1` à la liste d'exceptions du catch-all.

```c
if (op >= 0xF4E0 && op <= 0xF4FF &&
    op != 0xF4E1 && op != 0xF4E4 && op != 0xF4EB) {
    return consumed + s->lk_used;
}
```

### 2. SILICON-BOOT-REDIRECT removal — `calypso_c54x.c:9082-9094`

Le redirect statique PC=0xFF80 → 0x7120 (gated sur SP=0x1100 silicon-reset)
**bypass-ait l'init code à 0x7000-0x707F**. Cette zone init load A et
d'autres registres avant les BACC/CALA. Sans elle :
- 0x7127 contient `BACC A` (op=0xF4E2)
- A reste à 0 (silicon reset)
- BACC A jump à PC=0
- DSP boucle 0x0000-0x0080 boot stub

**Avec IDLE 1 fix + redirect retiré** : DSP boot via PROM1 reset vector
naturel à 0xFF80 (`4×RET` puis `CALL pmad 0x9F87` à 0xFF84). Atteint
PROM0 init à 0x960F (RET pop'd) à insn=15 **avec INTM=0**.

### 3. BSP IRQ rate gate — `calypso_bsp.c`

Sites BSP qui fire INT3 (vec 19) et BRINT0 (vec 21) sans gater
provoquaient `iota pending queue full` warnings (1963 events sur 9s).

**Fix** : gate par `IFR` avant chaque fire — skip si bit IRQ déjà set
(= DSP pas encore consommé le précédent) :

```c
if (bsp.dsp && !(bsp.dsp->ifr & (1 << 3))) {
    c54x_interrupt_ex(bsp.dsp, 19, 3);  /* INT3 */
}
```

3 sites patchés (2 pour INT3, 1 pour BRINT0 — les 2 autres BRINT0 avaient
déjà ce pattern).

Résultat : iota WARN 1963 → 634 (-68%).
